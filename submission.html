Project Overview

LOCKDOWN is a web-based escape room 3D experience built with Three.js. Players must solve three different puzzles to complete the escape.

Development Purpose

This project showcases the capabilities of Web 3D technology, particularly in interactive experience design. By integrating HTML5, CSS3, JavaScript, and 3D technologies, I've created an immersive user experience.

Technical Implementation

The project uses Three.js for 3D rendering, GSAP for animation effects, and JavaScript for interactive functionality. The page design employs responsive layout, adapting to different device sizes.

Development Process

This project was developed as an extension of the concepts learned in the laboratory sessions, but with significant original additions and creative direction:

  • Original Theme and Concept: Instead of following the Coca-Cola theme from lab work, I created an escape room experience with unique puzzle mechanics. This decision was made to create a more engaging and interactive experience that would better showcase my skills in 3D modeling and JavaScript interactivity.
  • Enhanced 3D Modeling: Created three detailed 3D models (Treasure Chest, Sequence Switches, and Directional Puzzle) with complex geometries, advanced materials, and custom animations. I deliberately chose models with varying complexity levels to demonstrate a range of 3D modeling techniques.
  • Expanded Interactivity: Implemented advanced JavaScript for multiple interaction types, including password entry, sequence matching, and puzzle rotation mechanics. Each interaction type demonstrates a different aspect of JavaScript-to-3D integration capabilities.
  • Custom Lighting System: Designed a dynamic lighting system with intensity controls beyond what was covered in the labs. This enhances the atmospheric effect of the escape room and showcases technical understanding of Three.js lighting.
  • Original UI Design: Created a custom user interface with a dark theme and responsive layout designed specifically for the escape room concept. The dark theme was chosen to enhance the immersive experience while improving visual contrast for interactive elements.

Technical Challenges & Solutions

During development, several technical challenges were encountered and solved:

  • 3D Model Integration: Creating seamless integration between 3D models and HTML/CSS interface required careful management of event listeners and raycasting techniques to detect user interactions with specific model parts.
  • Animation Synchronization: Synchronizing sound effects with animations required creating custom event systems to ensure timing precision and appropriate user feedback.
  • Responsive 3D Rendering: Maintaining performance across different device sizes required implementing dynamic camera positioning and adaptive rendering quality.
  • Model Complexity Management: Balancing model detail with performance led to implementing level-of-detail adjustments based on user interactions and viewing distance.

Performance Optimization

Several optimization techniques were implemented to ensure smooth performance:

  • Model Geometry Optimization: Reduced polygon counts on non-essential model parts while maintaining detail on focal interaction points.
  • Texture Compression: Implemented efficient texture compression to reduce loading times without sacrificing visual quality.
  • Code Efficiency: Used object pooling for particles and optimized render loops to maintain high frame rates during complex animations.
  • Asset Loading Strategy: Implemented progressive loading of assets to ensure users can begin interacting while heavier resources continue loading.
  • Memory Management: Carefully disposed of unused 3D objects and textures to prevent memory leaks during extended sessions.

3D Model Source Files

The original Blender source files for the 3D models are included in the GitHub repository and submission package. These files contain the complete developmental workflow including modeling, rigging, and texture mapping processes. The files can be found in the following locations:

  • Treasure Chest: models/treasurechest/source/treasure_chest.blend - This complex model includes animated hinges and lid movement with 4,862 polygons.
  • Sequence Switches: models/switches/source/switches.blend - Features interactive button mechanics with color-changing materials, comprising 3,245 polygons.
  • Directional Puzzle: models/puzzle/source/directional_pad.blend - Incorporates rotational animation and positional tracking, built with 2,783 polygons.

These Blender files contain the complete 3D modeling workflow, including materials, textures, and animation rigging used to create the final GLB files used in the web application. The models were carefully optimized for web delivery while maintaining visual fidelity.

Design Decisions

The project's design was guided by several key principles:

  • Immersive Experience: Dark color scheme enhances focus on the 3D models and creates an atmospheric escape room feel. The high contrast between interactive elements and backgrounds improves usability.
  • Intuitive Navigation: Simplified navigation structure ensures users can easily access all features without confusion. The persistent header ensures navigation remains accessible throughout the experience.
  • Progressive Disclosure: Information and interactions are presented in stages, preventing cognitive overload while maintaining engagement. Each puzzle reveals complexity as users engage with it.
  • Responsive Adaptability: The fluid layout system was designed to maintain functionality across devices without compromising the interactive experience. Control elements resize and reposition based on screen size.
  • Consistent Feedback: All user actions receive immediate visual and auditory feedback to reinforce interactions and guide users through the experience.

Testing Process

Thorough testing was conducted throughout the development process to ensure functionality and user experience:

  • Cross-browser Testing: Verified functionality in Chrome, Firefox, and Edge browsers to ensure compatibility.
  • Responsive Design Testing: Tested on various screen sizes (desktop, tablet, mobile) to confirm responsive layout works properly.
  • 3D Model Performance: Optimized model complexity and textures based on performance testing to ensure smooth rendering.
  • User Interface Testing: Conducted user tests to improve button placement and interaction feedback mechanisms.
  • Accessibility Testing: Verified color contrast ratios and added appropriate text alternatives for visual elements.

Testing results directly influenced the project development:

  • Reduced polygon count in 3D models to improve loading times
  • Improved lighting controls based on user feedback
  • Added visual feedback for interactive elements
  • Enhanced mobile layout for better touch interactions
  • Implemented progressive loading for better performance
